Title, Summary and Team Members

The title of our project is Water Simulation via Smoothed-Particle Hydrodynamics. Smoothed-particle hydrodynamics is a simulation method that can accurately simulate the behavior of continuous mediums. This method will be implemented to accurately simulate the motion of water, and ideally extend it to simulate the dynamics of other liquids of varying densities. The team members of this project are Nithin Tatikonda, Samantha Huang, Shreyas Patel, Xiaowen Yuan.


Problem Description

We are trying to realistically simulate liquids in this project using the Langrangian method. This is important as we want to be able to generate fluids with realistic motion for use in software such as movies and games. In general,we are going to try to solve this problem by using Project 4’s renderer to model liquid particle motion by implementing the physics for collisions similar to how we did for project 4. We will also try to implement smooth particle hydrodynamics.


Goals and Deliverables

What we plan to deliver

The baseline for our deliverables is to create simulations for water with realistic physical properties, such as gravity, tension, and viscosity. These goals can mostly be accomplished by using the information and papers presented in class as well as some additional resources, possibly for speeding up the calculations. We will be simulating liquid using a technique called smoothed particle hydrodynamics. The goal is to implement density constraints, surface tension, XSPH viscosity and other components of the technique. We will produce a clip of some fluid being dropped into or being emitted into an enclosed area or some object benign dropped into a fluid. We will also present water in different states, such as contaminated water with different viscosity. We’re not creating an interactive system. We will measure the quality of our system by how realistic the simulation is and how fast it can generate the pictures. We want to see how changing the different components of the technique will change the quality of the generated images. And we want to see if it is how to simulate liquids with different viscosity.

What we hope to deliver

We would want to experiment other types of fluids and more properties of water.


Schedule

Week 1

Designing a plan for how to represent liquid particles. Find a list of the equations needed to implement the motion and properties of liquid and figuring out how to translate them to code (ie which method of integration to use if necessary)

Week 2

Implementing the necessary data structures to represent the particles and implementing the code for the particle physics.

Week 3

Creating the shader files for best mimic the appearance of water and implementing the code for possible reflection or transparency of the water particles

Week 4

Debugging minor visual issues and attempting to speed up parts of the simulation code.
